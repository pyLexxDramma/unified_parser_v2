## 🎯 Описание проекта

Web Scraper — это полнофункциональное веб-приложение для автоматизированного сбора данных о компаниях из популярных картографических сервисов России. Система позволяет:

- Парсить отзывы, рейтинги и ответы компаний из Яндекс.Карт и 2GIS
- Анализировать данные по всей России или по конкретным городам
- Генерировать детальные отчеты в различных форматах
- Отслеживать прогресс парсинга в реальном времени
- Получать уведомления о завершении парсинга на email

## ✨ Основные возможности

### Парсинг данных

- **Множественные источники**: Поддержка Яндекс.Карт, 2GIS или обоих источников одновременно
- **Географический охват**: 
  - Парсинг по конкретному городу (быстрый режим)
  - Парсинг по всей России (200+ крупнейших городов)
- **Детальная информация**:
  - Название компании и адрес
  - Рейтинг и количество отзывов
  - Классификация отзывов (положительные, нейтральные, негативные)
  - Тексты отзывов с датами
  - Ответы компаний на отзывы
  - Расчет среднего времени ответа компании

### Генерация отчетов

- **CSV**: Структурированные данные для анализа в Excel/Google Sheets
- **JSON**: Полные данные со всеми деталями для программной обработки
- **PDF**: Красиво оформленные отчеты с агрегированной статистикой и детальными данными

### Веб-интерфейс

- Интуитивно понятный интерфейс с валидацией данных
- Модальное окно подтверждения перед запуском парсинга
- Отслеживание прогресса в реальном времени
- Детальная статистика по завершении парсинга
- Уведомления о возможных несоответствиях с реальными страницами

## 📊 Особенности точности данных

### Важная информация о точности

**Данные по 2GIS являются более точными и полными:**
- 2GIS находит в среднем в **2-4 раза больше отзывов**, чем Яндекс.Карты
- Обеспечивает более полное покрытие данных
- Более точные расчеты среднего времени ответа благодаря более полной выборке данных

### Возможные несоответствия

При сравнении данных в отчете с реальными страницами могут быть небольшие расхождения по следующим причинам:

1. **Отзывы vs Оценки**: На страницах может отображаться количество **оценок** (ratings), которое отличается от количества **отзывов** (reviews). Отзывы — это оценки с текстом. В отчете учитываются именно отзывы с текстом.

2. **Динамическая загрузка**: Отзывы могут загружаться динамически при прокрутке. Парсер загружает все доступные отзывы, но если требуется дополнительная прокрутка, некоторые отзывы могут быть не учтены.

3. **Фильтры на страницах**: На реальных страницах могут быть активны фильтры (например, "только положительные" или "только с ответами"), которые скрывают часть отзывов. Парсер собирает все отзывы без фильтров.

4. **Несколько карточек**: Агрегированная статистика показывает сумму отзывов из всех найденных карточек. Количество отзывов в отдельной карточке может отличаться от агрегированного значения.

5. **Обновление данных**: Данные на страницах обновляются в реальном времени. Между моментом парсинга и проверкой могут появиться новые отзывы.

### Рекомендации

- Для максимальной точности используйте **2GIS** или **оба источника**
- Для точной проверки откройте URL карточек в браузере и сравните данные вручную
- Учитывайте, что небольшие расхождения могут быть связаны с особенностями отображения данных на страницах сервисов

## 🛠 Технологии

### Backend

- **Python 3.7+**: Основной язык программирования
- **FastAPI**: Современный веб-фреймворк для создания API
- **Uvicorn**: ASGI-сервер для запуска FastAPI приложения
- **Selenium**: Автоматизация браузера для парсинга динамических страниц
- **BeautifulSoup4**: Парсинг HTML-контента
- **ReportLab**: Генерация PDF-отчетов
- **Pydantic**: Валидация данных

### Frontend

- **Jinja2**: Шаблонизатор для HTML
- **Vanilla JavaScript**: Клиентская логика без фреймворков
- **CSS3**: Стилизация интерфейса

### Дополнительные библиотеки

- **webdriver-manager**: Автоматическое управление драйверами браузера
- **requests**: HTTP-запросы
- **python-multipart**: Обработка форм
- **slowapi**: Rate limiting
- **psutil**: Мониторинг системных ресурсов

## 🚀 Установка и запуск

### Требования

- Python 3.7 или выше
- Google Chrome или Chromium
- Интернет-соединение
- 2-4 GB свободного места на диске (для отчетов и логов)

### Windows

1. **Клонируйте репозиторий**:
   ```bash
   git clone https://github.com/pyLexxDramma/unified_parser_v2.git
   cd unified_parser_v2
   ```

2. **Создайте виртуальное окружение**:
   ```bash
   python -m venv venv
   venv\Scripts\activate
   ```

3. **Установите зависимости**:
   ```bash
   pip install -r requirements.txt
   ```

4. **Настройте конфигурацию** (опционально):
   - Отредактируйте `config/config.json` при необходимости
   - Настройте переменные окружения в `.env` (если используется)

5. **Запустите сервер**:
   ```bash
   python run_server.py
   ```

   Или используйте PowerShell скрипт:
   ```powershell
   .\start_server.ps1
   ```

6. **Откройте браузер**:
   - Перейдите по адресу: `http://localhost:8000`
   - Войдите в систему (по умолчанию логин/пароль настраиваются в коде)

### Linux/macOS

1. **Клонируйте репозиторий**:
   ```bash
   git clone https://github.com/pyLexxDramma/unified_parser_v2.git
   cd unified_parser_v2
   ```

2. **Создайте виртуальное окружение**:
   ```bash
   python3 -m venv venv
   source venv/bin/activate
   ```

3. **Установите зависимости**:
   ```bash
   pip install -r requirements.txt
   ```

4. **Установите Chrome/Chromium** (если не установлен):
   ```bash
   # Ubuntu/Debian
   sudo apt-get update
   sudo apt-get install -y chromium-browser
   
   # macOS
   brew install --cask google-chrome
   ```

5. **Настройте конфигурацию** (опционально):
   - Отредактируйте `config/config.json` при необходимости

6. **Запустите сервер**:
   ```bash
   python run_server.py
   ```

7. **Откройте браузер**:
   - Перейдите по адресу: `http://localhost:8000`

### Docker (опционально)

Если у вас установлен Docker, вы можете использовать Dockerfile:

```bash
docker build -t web-scraper .
docker run -p 8000:8000 web-scraper
```

## 📖 Использование

### Заполнение формы парсинга

1. **Название компании**: Введите точное название компании, как оно указано на Яндекс.Картах или 2GIS
2. **Сайт компании**: Укажите домен компании (например: `example.ru` или `https://example.ru`)
3. **Источник**: Выберите Яндекс.Карты, 2GIS или оба источника
4. **Email**: Укажите email для получения уведомлений о завершении парсинга
5. **Область поиска**:
   - **Город**: Быстрый режим (несколько минут) - укажите конкретный город
   - **Страна / общий поиск**: Медленный режим (2+ часа) - выберите города из списка или оставьте все 200 городов
6. **Имя файла отчета**: Укажите имя для сохранения отчета (по умолчанию: `report.csv`)

### Запуск парсинга

1. Заполните все обязательные поля формы
2. Нажмите кнопку **"Поехали!"**
3. Проверьте данные в модальном окне подтверждения
4. Нажмите **"Запустить парсинг"** для начала парсинга или **"Редактировать введенные данные"** для изменения параметров

### Отслеживание прогресса

- После запуска парсинга вы будете перенаправлены на страницу статуса задачи
- На странице отображается:
  - Текущий статус парсинга
  - Прогресс выполнения
  - Время начала и длительность
  - Статистика по завершении

### Получение результатов

После завершения парсинга доступны:
- **CSV отчет**: Скачивание через веб-интерфейс
- **JSON отчет**: Полные данные для программной обработки
- **PDF отчет**: Красиво оформленный отчет с графиками и статистикой

## 🔧 Устранение неполадок

### Проблемы с запуском

- **Ошибка импорта модулей**: Убедитесь, что виртуальное окружение активировано и все зависимости установлены
- **Порт 8000 занят**: Измените порт в `run_server.py` или остановите процесс, использующий порт
- **ChromeDriver не найден**: `webdriver-manager` автоматически скачает драйвер при первом запуске

### Проблемы с парсингом

- **Парсинг не запускается**: Проверьте логи в `logs/parser.log`
- **Нет результатов**: Убедитесь, что название компании точно совпадает с данными на сайте
- **Медленный парсинг**: Это нормально для больших объемов данных. Парсинг по стране может занять 2+ часа

### Проблемы с отчетами

- **PDF не генерируется**: Проверьте права доступа к папке `output/`
- **Некорректные данные**: Проверьте логи парсинга и сравните с реальными страницами
